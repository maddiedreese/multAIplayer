# End-to-end coverage

Protocol v2 keeps MLS private state in the native Rust core. The browser build deliberately renders only `WebPreviewDemo`, so Playwright must not recreate the retired browser-side cryptography. A separate test-only UI-contract harness renders production React components and pure context builders for interaction coverage while labelling every simulated native or relay boundary on screen.

Security-critical journeys run at the narrowest layer that contains the real implementation:

| Journey                          | Executable coverage                                                                                                                                                                                                                                                                                                                                                                                                                                                                   | CI gate                             |
| -------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | ----------------------------------- |
| Invite approval and Welcome join | `e2e/invite-join.spec.ts` drives the production invite panel, composer, URL parser, and invite codec through explicit approve and deny decisions; it visibly labels relay transport and native KeyPackage/HPKE/Commit/Welcome handling as simulated. `apps/relay/test/process-security-journey.test.ts` remains the authoritative integration evidence: it drives the native MLS fixture through real relay HTTP, WebSocket, KeyPackage consumption, Add Commit, and Welcome routing. | `web-shell-e2e`, `security-journey` |
| Signed active-host handoff       | `e2e/host-handoff.spec.ts` drives the production handoff panel and room controls through offer, candidate request, explicit active-host approval, and exclusive control transfer with its relay and native MLS boundaries labelled as simulated. The native process journey separately publishes the signed handoff Commit, transfers MLS authority, and verifies the successor can publish the following removal/application epochs.                                                 | `web-shell-e2e`, `security-journey` |
| Codex turn approval              | `e2e/codex-turn-approval.spec.ts` drives member proposal, active-host-only approve/deny controls, bounded message and attachment previews, high-privilege context labels, bounded input construction, and execution UI transitions through the real approval card and context builders. Encrypted event delivery and native app-server execution are explicitly simulated; native authorization remains covered below the UI layer.                                                   | `web-shell-e2e`                     |
| Web preview isolation            | `e2e/web-shell.spec.ts` verifies the seeded preview cannot create, join, send, decrypt, persist private room material, or contact the relay                                                                                                                                                                                                                                                                                                                                           | `web-shell-e2e`                     |

A future native UI driver may add full macOS interaction journeys, but it must exercise the real Rust MLS and Codex app-server boundaries. The UI-contract harness proves rendered interaction and context-framing behavior only; its labelled simulations are not evidence for relay confidentiality, native authorization, or Codex app-server isolation.
