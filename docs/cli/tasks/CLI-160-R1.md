# CLI-160-R1: CLI Cargo package parser EOF correction

State: active

## Outcome

Correct the unsupported JavaScript `\z` escape in the CLI packaging parser so
the `[package]` section is parsed both before another Cargo section and at the
absolute end of the manifest, without changing any other packaging behavior.

## Allowed scope

- `apps/cli/release/release-lib.mjs`, limited to the absolute-end match.
- `apps/cli/release/release-policy.test.mjs`, limited to focused parser
  regression coverage.
- CLI-160-R1 status and verification evidence.

## Protected scope

- All product implementation, dependency, protocol, MLS, desktop runtime, and
  desktop release surfaces.
- CLI packaging behavior beyond the exact parser correction.
- Publication, Developer ID signing, release creation, and merges to `main`.

## Acceptance criteria

- [ ] ESLint accepts the parser expression without suppression or waiver.
- [ ] The CLI version is parsed when `[package]` is followed by another section.
- [ ] The CLI version is parsed when `[package]` ends at absolute end-of-input.
- [ ] Existing invalid-manifest failures and all other packaging policy remain
      unchanged.
- [ ] No manifest, lockfile, dependency, desktop, or protected release path is
      modified.

## Verification

- Focused parser and complete CLI packaging-policy tests.
- Full `npm run verify` and CLI-160 artifact build/verification.
- Classification/release-isolation, exact-path, ancestry, cleanliness, and
  protected desktop-release checks.

## Stop conditions

Stop for any broader parser rewrite, packaging-policy change, dependency or
manifest change, failing/skipped/non-executing required test, protected-path
change, release action, or unresolved security or release-isolation concern.
