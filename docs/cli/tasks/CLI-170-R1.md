# CLI-170-R1: Public CLI installation and release documentation

State: owner_approved
Plan version: 1.1

## Outcome

Provide a single-command, fail-closed installation path and make the CLI's
public-alpha documentation match the accepted implementation and release
boundaries.

## Dependencies

- CLI-160 and CLI-160-R1 accepted packaging.
- CLI-170 technical readiness evidence.
- Owner approval on 2026-07-19.

## Allowed scope

- `apps/cli/install.sh`, CLI release documentation, and installer policy tests.
- User-facing CLI quickstart, compatibility, limitations, update, and uninstall
  documentation.
- Semantic documentation checks and corrections to stale CLI documentation.
- `docs/cli/status.md` evidence for this correction.

## Protected scope

- Desktop release metadata, workflows, Cargo.lock, updater, signing,
  notarization, versions, and asset manifests.
- CLI publication, tags, releases, credentials, and signing identities.
- Product behavior, dependencies, relay contracts, MLS policy, and desktop
  behavior.

## Acceptance criteria

- [ ] The documented public installation is one copyable command.
- [ ] The installer supports only Apple-silicon macOS and fails closed unless
      checksums, Developer ID signature metadata, and Gatekeeper assessment pass.
- [ ] Installation does not weaken the independent CLI/desktop release boundary.
- [ ] Quickstart, compatibility, limitations, update, uninstall, and recovery
      guidance match the implemented CLI.
- [ ] Invite capabilities remain stdin-only in every maintained command example.
- [ ] Documentation checks reject the stale pre-implementation and unsafe join
      examples.

## Required verification

- `sh -n apps/cli/install.sh`.
- `node --test apps/cli/release/release-policy.test.mjs`.
- `npm run docs:check`.
- `node tools/ci/run-cli-checks.mjs`.
- Changed-path classification and exact protected desktop-release diff review.

## Stop conditions

Stop if this correction requires publication, a signing credential, a desktop
release change, a dependency, a wire/MLS change, or a product behavior change.
