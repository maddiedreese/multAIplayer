# CLI-150: Security hardening and threat-model update

State: active

## Outcome

Audit and harden the implemented CLI boundaries, update normative security
documentation to match reality, and close launch-blocking findings.

## Allowed scope

- CLI/shared security fixes, fuzz/property tests, diagnostics/redaction, threat
  model, and relevant ADRs.

## Acceptance criteria

- [ ] Secrets remain out of relay plaintext, logs, diagnostics, args, snapshots,
  crash reports, and artifacts.
- [ ] Terminal injection and trusted-prompt spoofing are blocked.
- [ ] Authority, path, origin, request, and session bindings fail closed.
- [ ] Corrupt state never triggers insecure regeneration or deletion.
- [ ] Security claims cite executed tests and describe residual risks accurately.

## Verification

- Fuzz/property suite, adversarial multi-process journeys, secret scan, native
  security tests, and owner review of threat-model changes.
