# CLI-120: Codex proposals, context, and hosted turns

State: waiting_for_orchestrator_approval

## Outcome

Allow any room member to submit an encrypted proposal and only the active host
to preview, approve, context-build, and run one local Codex turn.

## Allowed scope

- Proposal state machine, bounded context builder, host CLI prompts, and room
  assistant-message projection.

## Acceptance criteria

- [ ] Proposal and approval are exact-room, exact-ID, expiring, and idempotent.
- [ ] One active turn and at most one pending proposal initially.
- [ ] Host sees proposer, task, context extent, model, tier, reasoning, and sandbox.
- [ ] Host loss cancels safely and never reports success.
- [ ] Excluded secrets and raw protocol/app-server data cannot enter context.

## Verification

- Fixture and real-compatible app-server turn journeys.
- Context-bound, authority-loss, duplicate, expiry, and cancellation tests.
