# CLI-010-R1: Journey CI toolchain reliability correction

State: active

Owner authorization: 2026-07-19

## Outcome

Restore deterministic execution of the required native fuzz gates by pinning a
compatible nightly Rust toolchain and `cargo-fuzz` version, then prove the full
product-journey workflow is reliable without weakening any test or gate.

## Allowed scope

- The nightly Rust and `cargo-fuzz` installation lines in
  `.github/workflows/journeys.yml`.
- Focused CI comments or deterministic verification needed to make the exact
  toolchain pairing reviewable.
- CLI program status evidence for this corrective task.

## Protected scope

- Both existing native fuzz targets, their arguments, execution time, failure
  behavior, corpus handling, and artifact handling.
- The native-shell journey, artifact requirements, changed-domain
  classification, required-jobs aggregation, and every other workflow job.
- Product source, tests, dependencies, lockfiles, wire contracts, MLS policy,
  desktop runtime behavior, release workflows, signing, notarization, updater,
  versions, assets, and all other protected release surfaces.

## Acceptance criteria

- [ ] The existing `cargo-fuzz 0.12.0`/moving-nightly incompatibility is
      reproduced or established from the failed run evidence.
- [ ] Nightly Rust and `cargo-fuzz` are pinned to an explicit compatible pair.
- [ ] Both existing native fuzz targets actually execute and pass unchanged.
- [ ] The real two-client native MLS journey executes and passes unchanged.
- [ ] The complete `Product journeys` workflow passes twice on the exact
      corrective commit, including required artifact uploads.
- [ ] No gate, timeout, assertion, target, artifact requirement, or product
      behavior is weakened or bypassed.

## Verification

- Two complete successful `workflow_dispatch` runs of
  `.github/workflows/journeys.yml` on the exact corrective commit.
- Exact run, commit, job, step, fuzz-execution, native-journey, and artifact
  evidence for both runs.
- Existing changed-domain classification and release-isolation tests.
- Workflow syntax/action-pin review, exact changed-path audit, `git diff
  --check`, clean worktree, and protected desktop/release audit.
