# CLI-060: MLS client state, storage, and outbox

State: waiting_for_orchestrator_approval

## Outcome

Wrap the existing MLS engine in a UI-independent client service with atomic
state, outbox publication, acknowledgements, stale-epoch handling, and explicit
rejoin/corruption behavior.

## Allowed scope

- Additive shared/native client code and CLI adapters.
- Existing `mls-core` API additions that preserve policy and desktop behavior.

## Acceptance criteria

- [ ] No second MLS implementation or ciphersuite choice.
- [ ] State is durable before externally visible success.
- [ ] Publish/replay is idempotent across crash points.
- [ ] Corruption never silently regenerates or deletes secrets.
- [ ] Desktop MLS tests remain unchanged in meaning and pass.

## Verification

- Crash-point, stale-epoch, outbox replay, and corrupt-state tests.
- Full MLS workspace tests.
