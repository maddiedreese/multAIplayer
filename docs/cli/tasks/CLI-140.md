# CLI-140: Desktop/CLI interoperability journeys

State: complete

## Outcome

Prove desktop and CLI clients share rooms, chat, proposals, host turns, replay,
membership, and safe unsupported-event behavior over the same contracts.

## Allowed scope

- `e2e/cross-client/**`, compatibility adapters, and bug fixes that do not alter
  approved semantics without owner approval.

## Acceptance criteria

- [ ] CLI host/CLI participant.
- [ ] CLI host/desktop participant.
- [ ] Desktop host/CLI participant.
- [ ] Rooms and chat created by either client.
- [ ] Proposal approval by either supported host client.
- [ ] Mixed-client reconnect, replay, removal, and unsupported-event rendering.
- [ ] Host handoff involving CLI fails explicitly as unsupported.

## Verification

- Required mixed-client matrix on supported macOS hardware.
