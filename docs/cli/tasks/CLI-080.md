# CLI-080: Secure invite codes and host-mediated admission

State: waiting_for_owner_approval

## Outcome

Expose the existing capability-based MLS admission through versioned,
checksummed, high-entropy pasteable codes with host approval.

## Allowed scope

- CLI invite-code codec and existing invite/admission orchestration adapters.
- No weakening or replacement of current capability bindings.

## Acceptance criteria

- [ ] At least 128 bits of unpredictable capability material.
- [ ] Codes are redacted, revocable, expiring, and single-use initially.
- [ ] Host sees GitHub identity and device fingerprint before deciding.
- [ ] Approval/denial/expiry/replay are bound and idempotent.
- [ ] Desktop invite interoperability is preserved where the representation permits.

## Verification

- Two-device approve/deny/expire/replay journeys.
- Codec property/fuzz tests and secret-reflection tests.
