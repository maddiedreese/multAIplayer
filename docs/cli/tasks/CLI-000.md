# CLI-000: Program orchestration and governance

State: active_orchestrator

## Outcome

Maintain the approved plan and status ledger, monitor one active implementation
task, independently verify completion, merge accepted work only into
`codex/cli-integration`, and approve the next dependency-ready task automatically.

## Allowed scope

- Read access to every CLI task thread, worktree, branch, pull request, test, and
  repository file needed for verification.
- Status/orchestration documentation updates.
- Safe merges into `codex/cli-integration` only.
- Explicit approval messages to one dependency-ready task at a time.

## Acceptance criteria

- [ ] At most one implementation task is active.
- [ ] The active task starts from the latest accepted integration baseline.
- [ ] Completion is verified from diff, tests, task criteria, and release safety;
  an agent's completion claim alone is insufficient.
- [ ] Accepted work is merged only into `codex/cli-integration`.
- [ ] Status and evidence are updated before the next task starts.
- [ ] The next task has completed dependencies and receives exact approval.
- [ ] Stop conditions pause automation and notify the owner.

## Required verification

- Task-specific focused and composed checks.
- Clean integration diff and repository status.
- Desktop release-boundary audit for every task.
- Dependency and status-ledger consistency.

## Prohibited actions

- Merge to `main` or publish any release.
- Change the approved plan, locked decisions, crypto policy, or wire contract.
- Waive tests, accept unexplained failures, or run overlapping writers.
- Answer a product/security question reserved for the owner.
- Advance after a protected-scope, verification, authority, or integration doubt.

## Heartbeat behavior

Every 15 minutes, follow `docs/cli/orchestration.md`. If a task is active,
inspect progress without duplicating its work. If complete, verify and integrate
or pause. If no task is active, approve the next dependency-ready task. If all
tasks are complete, disable further advancement and notify the owner.
