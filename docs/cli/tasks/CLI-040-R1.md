# CLI-040-R1: Device identity reliability correction

State: active

Owner authorization: 2026-07-18

## Outcome

Reproduce and correct intermittent CLI device identity generation or
crypto-provider initialization failures without changing any identity contract.

## Allowed scope

- CLI device identity generation and narrowly required crypto-provider
  initialization in existing CLI/platform adapters.
- Focused tests and deterministic test instrumentation for identity reliability.

## Protected scope

- Identity account names, storage versions, public/private key formats, device
  registration fields, relay wire formats, and desktop interoperability contracts.
- MLS cryptographic policy, ciphersuite selection, dependencies, workspace
  boundaries, desktop runtime behavior, and every protected release surface.

## Acceptance criteria

- [ ] The intermittent failure has an evidence-backed root cause.
- [ ] The correction preserves signing/HPKE identity bytes and registration
      compatibility.
- [ ] Failure paths remain redacted, durable, and fail closed without deleting or
      replacing previously durable identity material.
- [ ] Repeated serial and parallel identity generation/restoration runs pass.

## Verification

- Repeated serial and parallel reproduction/regression runs.
- Complete locked CLI verification.
- Complete `mls-core` verification and durable identity lifecycle tests.
- Focused desktop identity/device registration compatibility and desktop build
  or typecheck gates applicable to the changed boundary.
- Dependency, changed-path, release-isolation, cleanliness, and protected-path
  audits.
