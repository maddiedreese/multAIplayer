# CLI-130: Privileged approvals and shared activity

State: waiting_for_owner_approval

## Outcome

Render supported Codex server requests only to the active host, bind each human
decision exactly, and share only the normalized bounded activity projection.

## Allowed scope

- CLI approval adapter, normalized activity rendering, and shared projection
  fixtures that preserve the existing contract.

## Acceptance criteria

- [ ] Command, file, permission, tool-input, and supported elicitation requests
  require exact host approval.
- [ ] Cross-room/session/request responses and non-host responses are rejected.
- [ ] Unknown, malformed, expired, shutdown, and authority-lost requests fail closed.
- [ ] No `--yes`, unattended host, or prompt spoofing path exists.
- [ ] Raw credentials, environment, arbitrary output, and unknown upstream fields
  are never shared.

## Verification

- Request matrix, expiry, cancellation, cross-room/session, redaction, and
  terminal-spoof tests.
