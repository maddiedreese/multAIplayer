# Design and historical context

This directory holds useful but non-normative material that should not become
merge ceremony: superseded design records, retrospective rationale recovered from
earlier hardening work, and condensed project history. These documents are
maintained by hand, are not generated evidence, and are not CI gates.

Current sources of truth remain outside this directory:

- [Threat model](../threat-model.md) for security claims and residual risks;
- [architecture decisions](../decisions/README.md) for active design constraints;
- [self-hosting](../self-hosting.md) for operator procedures; and
- [CONTRIBUTING](../../CONTRIBUTING.md) for development and merge expectations.

Historical material can explain why a design changed, but it must link back to
the current source rather than restating present policy.
